Objective

Add lazy loading support to hero images so that only the first image is loaded on page load, and the others are loaded on demand only. This should result in quicker page load times.

Built-In Refrigeration

Performance Comparison

* Results shown are a before and after adding lazy loading support to this current page, using 3 images.

Page Weight Document Loaded Fully Loaded
Original 2.1mb 605ms 1.67ms
Lazy Loading 1.6mb 712ms 1.22s

Development Considerations

  • For the purpose of this demo I stripped out responsive image support, as the carousel used (Slick) and the responsive solution (Picturefill) we use don't work together right out of the box. Luckily Slick can tap into Picturefill's API to take care of this.
  • Carousels are set to autoplay. This was done in an effort to try and keep a page "fresh," but it also defeats the lazy loading, so it has been disable.
  • It appears that fade:true seems to break the lazy loading. Since autoplay is disabled, do we need this or is the native slide functionality fine, as seen in other content carousels across the site.

Conclusions

  • This solution is super effective in decreasing page load times and handling excess page weight on hero image heavy pages.
  • This could (and absolutely should) be implemented into store product detail pages, as well as all carousels on the site.
